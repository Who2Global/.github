# Support

This page explains how to get help with repositories owned by the Who2Global GitHub organisation.

## General support

For general questions, setup help or non-urgent issues, use the normal support route agreed with the Who2Global team.

GitHub issues should be used for clear repository-related work only, such as:

* Confirmed bugs
* Feature requests
* Documentation improvements
* Technical tasks
* Reviewable development work

## Before raising an issue

Before raising an issue, please check:

* The correct repository is being used
* A similar issue does not already exist
* The request is clear enough for someone else to understand
* No secrets, passwords, tokens or customer-sensitive data have been included

## Security issues

Do not raise public GitHub issues for security vulnerabilities.

Use the security policy instead.

## What to include

A good support request should include:

* A short summary
* What you expected to happen
* What actually happened
* Any relevant screenshots or error messages
* The affected environment, branch or service
* Steps to reproduce the issue, if applicable

## What not to include

Do not include:

* Passwords
* Tokens
* API keys
* Certificates
* `.env` files
* Production credentials
* Private customer data
* Sensitive personal information

## Repository-specific support

Some repositories may have their own support notes, runbooks or escalation paths.

Where repository-specific guidance exists, follow that guidance first.
