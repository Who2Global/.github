# Security Policy

Who2Global takes security issues seriously.

Please do not report security vulnerabilities through public GitHub issues, pull requests, discussions or comments.

## Reporting a vulnerability

If you believe you have found a security issue, report it privately to the Who2Global team.

Your report should include:

* A clear summary of the issue
* The affected repository, service or component
* Steps to reproduce the issue where safe to provide
* Any relevant screenshots, logs or error messages with secrets removed
* Your contact details for follow-up

## Do not include

Please do not include:

* Passwords
* Tokens
* API keys
* Certificates
* Private customer data
* Production credentials
* Sensitive personal information

## Public disclosure

Please do not publicly disclose a suspected vulnerability until Who2Global has had a reasonable opportunity to review, validate and respond.

## Scope

This policy applies to repositories owned by the Who2Global GitHub organisation unless a repository has its own more specific security policy.
